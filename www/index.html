Welcome to Kijaro

Kijaro provides an area for those interested in adding new language features to Java to try out their ideas.

The project contains a copy of the Open JDK javac compiler, which has been customised to add additional language features. We encourage everyone with a great idea to code it up in java and host it here.

News

JDK 7 will contain some new language features, described as 'small'. Kijaro is a useful proving ground and discussion centre for those ideas (with real implementations).

Features

Each feature is developed in a separate branch. The features currently being worked on are:

  • First Class Methods (download) - Method, Field & Constructor literals, Method references, Inner methods
  • Properties - Simple property declarations, literals and usage
  • Abstract Enums - Abstract superclasses for enums
  • Static Implements / Contracts - Implementing an interface using static methods
  • Anonymous parameters - Not specifying parameter names where they have no meaning
  • List comprehensions (download) - Python style list comprehensions
  • For-each loop for Maps (download) - Looping around maps
  • Array-like access for Lists and Maps - Using the [] operator to access lists and maps
  • Enhanced strings - Multi-line and Raw strings
  • Byte/Short literal suffixes and Short hex and binary literals (download) - Extended literals
  • Leniency - Uncaught checked exception as warning rather than error (download)

This effort is run independently of Sun, and no warranty is given on the quality of the changes made to javac. However, we do try and keep the code changes minimal, and as such we hope that all existing Java programs will still compile.

This project is similar to the Kitchen Sink Language (KSL), however there are a few differences. This project will not require code reviews by compiler experts, or the signing of legal documents. It is simply a place where those who want to change Java for the better can come together and demonstrate their idea for real.

If you have a great idea for a language change that you want to code, why not join us? Simply subscribe to the dev mailing list and let us know so we can open up a branch.

Rules

There are just a few rules to try and keep everything going:

  • Documentation. Each new language feature must have some form of associated document, even if its just a blog. It doesn't have to be much, but should have an outline of why the feature is needed and the syntax implications.
  • Backwards compatibility. On svn trunk all existing Java code must compile.
  • Comments. Each change in svn trunk must have a comment so we can find it later, such as 'FCM-MREF'.
  • er, thats about it!

Developer information

Kijaro is based on Sun OpenJDK javac, currently from jdk7-b23, which is contained in a vendor branch. Unless the OpenJDK version contains major changes that we need, it is unlikely that this version will be updated in the near future.

Each subproject will use a separate branch. Subversion trunk will contain the merged changes.

The compiler is built using the standard ant script in the make directory. To get this to work, you can follow these build instructions.

The resulting compiler can be used in place of a normal compiler. The compiler build will create a javac unix shell file. Windows users can use this batch file.

IDE integration files may be checked into subversion, and we would welcome such submissions. We would also greatly welcome anyone with experience of integrating syntax changes into IDEs.

Information on making changes to the compiler:

  • Detailed tutorial, based around adding an operator
  • Summary of the AST and adding a new node

Subversion also contains the source for all the website pages. The website must only be edited using subversion. It must not be edited using the online java.net tool.
